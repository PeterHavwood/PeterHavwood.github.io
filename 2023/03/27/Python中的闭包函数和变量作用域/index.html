变量作用域

Python中变量作用域分为：“局部（local）作用域”、“嵌套作用域”、“全局（global）作用域”和“内置作用域”。只有模块（module），类（class）以及函数（def、lambda）才会引入新的作用域，其它的代码块（如 if/elif/else/、try/except、for/while等）是不会引入新的作用域的。

上面代码中的counter定义在if语句中，它是一个全局变量。而countFun函数中的count就是一个局部变量，它属于该函数的局部作用域中，在该函数之外是不能访问它的。但是对于嵌套在countFun函数中的fun函数来说，count则属于嵌套作用域，我们可以访问它，但是count对fun函数而言并不是一个局部变量。所谓的内置作用域是Python内置的那些标识符，如print、def等。Python编译器会按照“局部作用域”、“嵌套作用域”、“全局作用域”和“内置作用域”顺序查找一个变量并确定其作用域。

对于嵌套函数中的那些嵌套作用域中的变量（也就是嵌套函数外面一层的函数中的变量）来说，我们可以在嵌套函数中读取它的值，但是一旦涉及到赋值操作（也就是修改它的值），那么编译器会把这个变量视为一个局部变量。如果上面的代码中我们没有nonlocal count语句，那么编译器就会把count视作fun函数的一个局部变量，而fun函数中并没有声明这个变量，才会出现‘local variable ‘count’ referenced before assignment’的错误信息。我们在代码中为count加上nonlocal关键字，就是告诉编译器，count不是一个局部变量，而是来自countFun的嵌套变量。只有这样，我们才能在闭包函数中修改countFun的count变量的值。

还有一个关键字global，当我们需要在一个函数内部声明一个全局变量时，可以使用它来修饰。

也就是说，如果希望在嵌套函数中修改函数外的嵌套变量，需要nonlocal关键字；如果需要在函数中声明的变量成为全局变量，在函数外也可以被访问和修改，则应该使用global关键字。
